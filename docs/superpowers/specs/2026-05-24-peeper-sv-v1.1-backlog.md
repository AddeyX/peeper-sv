### 1. `peeper.config.ts` loader

**Problem:** Spec Â§3 promises `.ts` config files. `src/server/load-config.ts:34` currently throws:


### 2. Viewport picker missing `custom`

**Problem:** Spec Â§10 lists `natural | mobile 375 | tablet 768 | desktop 1280 | custom`. `src/gallery/lib/ViewportPicker.svelte` ships only the first four.

**Fix:** Add a fifth option `{ name: "custom", width: -1 }` (or similar sentinel). When selected, reveal a number input bound to a separate `customWidth` state. Wire into `Preview.svelte`'s viewport width binding. Extend `tests/e2e/gallery.spec.ts` with a custom-viewport test.

### 3. FS watcher recovery + manifest-failure banner

**Problem:** Spec Â§11 specifies two behaviors that are not wired:

- "FS watcher dies â†’ re-create watcher once, then log + manual reload"
- "Manifest virtual module fails â†’ plugin returns stub `{ components: [] }` + logs, gallery shows 'Scan failed: <reason>' banner"

`src/plugin/index.ts` does not wrap `rebuildManifest()` in a top-level try; an uncaught throw in `scan()` crashes the buildStart hook. The watcher has no `error` handler.

**Fix:**

- Wrap `rebuildManifest()` body in try/catch. On failure, set `state.manifest = { generatedAt: Date.now(), components: [], scanError: <message> }` and log to stderr. Extend `Manifest` type to include optional `scanError?: string`.
- Add `Sidebar.svelte` / `App.svelte` banner that renders when `manifest.scanError` is present.
- Add `watcher.on("error", ...)` handler that calls a `recreateWatcher()` helper once, then if it fails again, sets a state flag and logs.

### 4. Stale JSDoc on `ComponentEntry.importPath`

**Problem:** `src/shared/types.ts:25` doc comment says `importPath: "/src/lib/Button.svelte"`. Actual value is `/@fs/<absolute-path>` (changed in T17 to work with Vite's filesystem serving).

**Fix:** Update comment to `"/@fs/<absolute filesystem path>, suitable for Vite dynamic import in browser"`.
